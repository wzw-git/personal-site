1. 系统概览

本站为「静态前端 + REST API + 关系型数据库」的经典三层结构：MySQL 8 持久化用户、文章、标签；Node.js（Express） 提供 /api/* 接口（鉴权、CRUD、上传、天气代理）；Nginx 托管 public/ 下的 HTML/CSS/JS，并将 /api/ 反向代理到 API 容器。三者由 Docker Compose 编排，便于单机一键启动与迁移。

重要：Nginx 只负责静态文件与反代，不能替代 MySQL。数据库始终由独立容器（或外部 MySQL 实例）运行。

6. 部署操作步骤

  1. 准备环境：安装 Docker Engine 与 Docker Compose 插件；克隆本仓库到服务器或本地。
  2. 配置密钥：复制 .env.example 为 .env，填写 MYSQL_*、JWT_SECRET（长随机串）、ADMIN_PASSWORD 等；切勿将含真实密钥的 .env 提交到 Git。
  3. 启动：在仓库根目录执行 docker compose up -d --build。
  4. 验证：浏览器访问 http://主机:HTTP_PORT（默认 8090，见 .env 中 HTTP_PORT）；打开 /api/health 应返回 {"ok":true}。
  5. 更新代码：拉取新代码后执行 docker compose up -d --build；若仅改 API，可 docker compose up -d --build api。
  6. 生产加强：为 API 容器设置 NODE_ENV=production；配置 HTTPS 终端（CDN / Nginx / Caddy）并启用 HSTS；按需设置 CORS_ORIGIN。详见下文「安全」与仓库根目录 SECURITY.md。

7. 环境变量说明

以下变量多由根目录 .env 注入 Compose（见 docker-compose.yml）。本地直连 Node 时可使用 server/.env（参考 server/.env.example）。

变量 说明
HTTP_PORT 宿主机映射 Nginx 80 的端口（默认 8090）。
MYSQL_ROOT_PASSWORD / MYSQL_PASSWORD 数据库 root 与应用用户密码；上线必须修改默认值。
JWT_SECRET 签名 JWT 的密钥，越长越随机越好；生产环境且 NODE_ENV=production 时，使用弱/占位密钥将导致 API 拒绝启动。
ADMIN_USERNAME / ADMIN_PASSWORD 空库首次启动时创建的初始管理员；已有数据不会自动改密。
CORS_ORIGIN 可选。逗号分隔的允许源（完整 URL）；限制浏览器跨域调用 API。同域部署可不设。
NODE_ENV 设为 production 时启用更严限流与 JWT 强密钥校验（建议在 API 服务上设置）。
BLOG_TITLE / BLOG_DESCRIPTION 站点标题与 meta 描述（接口 /api/site）。
SITE_OWNER_NAME 顶栏时段问候用显示名（可留空）。
JUHE_WEATHER_KEY 等 首页天气；密钥仅服务端使用。缓存与每日额度见 JUHE_WEATHER_DAILY_LIMIT、WEATHER_CACHE_TTL_MS。
UPLOAD_DIR API 容器内上传目录；Compose 中通常挂载到 ./public/uploads。

8. 安全与生产检查

  • 服务端：参数化 SQL、JWT 固定 HS256、Helmet 部分头、登录/注册/上传/写文限流、上传文件魔数校验、文章与标签长度/数量校验。
  • Nginx：X-Frame-Options、X-Content-Type-Options、Referrer-Policy、Permissions-Policy；禁止访问 /uploads/ 下以 . 开头的路径（含天气状态文件）。
  • Markdown 正文经 DOMPurify 净化后渲染；作者属于信任边界，仍建议勿粘贴不可信来源的复杂 HTML。
  • 完整清单、残余风险与报告方式见仓库根目录 SECURITY.md。

10. 仓库目录结构

  • docker-compose.yml — 服务定义、环境变量、卷、健康检查。
  • nginx/default.conf — 静态根目录、/api/ 反代、安全头、上传体积限制、隐藏文件规则。
  • server/ — API 源码、Dockerfile、package.json、schema.sql、迁移脚本。
  • public/ — 前端页面、styles.css、js/、uploads/（用户上传，勿清空生产数据时误删）。
  • README.md — 仓库首页说明与快速开始（GitHub 展示）。
  • docs/MIGRATION.md — 换云服务器迁移：备份、恢复、域名与 HTTPS。
  • scripts/backup-for-migration.sh — 迁出前一键导出数据库与打包 uploads。
  • 部署说明.txt — 纯文本快速部署与安全提示。
  • SECURITY.md — 安全专项说明。

11. 本地开发 API（不经过 Docker）

安装 Node 18+ 与可访问的 MySQL，在 server 目录执行 npm install，复制 server/.env.example 为 server/.env 并填写连接信息，运行 npm start。前端可将 public/js/api-base.js 中的 API 根地址指向本地端口进行联调（注意 CORS 与同源策略）。

12. 常见问题

  • Cannot POST /api/auth/register：多为 API 镜像未重建，执行 docker compose up -d --build api。
  • 502 Bad Gateway：API 未就绪或崩溃；查看 docker compose logs api。
  • 生产启动失败（JWT）：检查是否设置 NODE_ENV=production 却仍使用示例 JWT_SECRET；改为强随机串。
  • 天气不显示：未配置 JUHE_WEATHER_KEY 或上游额度用尽；接口会返回说明 JSON。
  • docker 权限：需要 sudo 或将用户加入 docker 组。
